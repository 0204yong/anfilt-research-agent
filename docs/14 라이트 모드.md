# 14 라이트 모드

> 상태: **구현·라이브 e2e 검증 완료 (2026-07-11).**
> 배경: "연동된 LLM API 비용이 너무 부담된다"는 사용자 의견(2026-07-11) +
> `ESG_에이전트_업무패턴_분석.docx`(사용자 업무 패턴 분석).

## 핵심 아이디어 — 지식창고 우선(Knowledge-first)

풀 파이프라인은 실행당 `프로바이더 수 × (1+라운드) + 2`회의 대형 호출을 쓴다.
그런데 사용자의 일상 업무 대부분은 "새로운 심층 조사"가 아니라 **반복 프레임의
재실행**(검증 4단계, 전년도 자료 재사용 등)이다. 라이트 모드는 호출 순서를
뒤집는다:

| 단계 | 내용 | 비용 |
|---|---|---|
| 0 | 지식볼트에서 관련 엔티티 검색·주입 (→ [[13 지식 볼트와 온톨로지]] 4단계 재사용) | 무료 |
| 1 | **경량 모델 1~2회 호출** — 키워드 있으면 웹 조사 1회 + 구조화 1회, 없으면 구조화 1회 | 풀 모드의 수십분의 1 |
| 2 | 기존 훅이 아카이브(ra_runs)·온톨로지(ra_vault)에 자동 축적 | 경량 1회 |

**쓸수록 0단계 적중률이 올라가 비용이 더 내려가는 구조**가 세일즈 포인트.
토론·채점이 없으므로 신규 고객사·고위험 주제의 첫 조사는 풀 모드 권장
(UI 도움말에 명시).

## 구현 (2026-07-11)

- **`core/light.py`** (Streamlit 비의존) — `run_light_pipeline(provider, brief,
  target_pages)`: 키워드 있으면 `_research_prompt` 웹 조사 → `_structure_prompt`
  로 REPORT_SCHEMA 구조화. **풀 파이프라인과 동일한 `PipelineResult` 반환** →
  결과 5탭·보고서 3종·아카이브·볼트 내보내기 전부 무수정 재사용.
  조사 원문은 findings[0]으로 들어가 "개별 조사 결과" 탭에 표시
- **경량 모델 구성** — `config.LIGHT_MODEL_DEFAULTS` + `<KEY>_LIGHT_MODEL` env
  오버라이드, `build_providers(light=True)`. 기본값: Gemini `gemini-3.5-flash` /
  Claude `claude-haiku-4-5-20251001` / GPT `gpt-5-mini`(키 등록 후 검증 필요).
  ⚠️ `gemini-2.5-flash`는 신규 사용자에게 404 (2026-07 실측 — ListModels에는
  나오지만 generate가 거부됨)
- **업무 템플릿 5종** (`LIGHT_TEMPLATES`) — 업무패턴 분석 2-2("검증 업무는 고정
  체크리스트") 반영: 자유 조사 / 보고서 검증 4단계 / 공시기준 부합 검토 /
  LCA·PCF / 기후 시나리오·재무영향. 선택하면 instructions에 고정 프레임 자동
  append — 매번 같은 지시를 다시 쓰지 않아도 됨
- **app.py** — 사이드바 최상단 "실행 모드" 라디오(**라이트가 기본값**).
  라이트: 단일 경량 LLM 선택(gemini 우선)·분량 기본 6장, 토론/채점/종합방식
  컨트롤 숨김. 실행 파라미터에 `mode: "light"`·`model`·`template` 기록 →
  재열람 시 그대로 복원. **온톨로지 추출도 해당 run의 모드를 따라 경량 모델
  사용** (훅이 사이드바 상태가 아니라 run params 기준으로 프로바이더 재구성)

## 비용 비교 (동일 주제 기준)

| | 풀 모드 (3사·1라운드) | 라이트 모드 |
|---|---|---|
| 호출 수 | 8회+ (조사3·토론3·채점1·종합1) + 추출 1 | 1~2회 + 추출 1 |
| 모델 | 프런티어(Pro/Opus/GPT-5) | 경량(Flash/Haiku/mini) |
| 체감 비용 | 기준 | **~1/30 이하** (호출 ×1/4, 단가 ×1/10 수준) |

## 라이브 e2e (2026-07-11)

CBAM 전환기간 주제 + 키워드 → 축적 지식 4건 주입(0단계) → Flash 웹 조사 +
구조화(1단계, 2회 호출) → 보고서 5탭 렌더(출처 4건) → 아카이브 저장
(params: mode=light·model=gemini-3.5-flash·template=free) → 온톨로지 반영
신규 1·갱신 3(2단계, 추출도 Flash) 전 구간 실측 통과.
