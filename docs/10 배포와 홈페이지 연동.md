# 10 배포와 홈페이지 연동

회사 홈페이지(anfilt-homepage.netlify.app)에 Research Agent를 **제품으로 등록**하고
실행 가능하게 만드는 작업. 실무 절차는 저장소 루트 [[../DEPLOY]](`DEPLOY.md`) 참고,
이 문서는 설계 배경과 구조를 기록한다.

## 핵심 제약: Streamlit ≠ Netlify

- Research Agent는 **파이썬(Streamlit) 서버 앱** — 지속 실행되는 WebSocket 서버가 필요.
- Netlify는 **정적 호스트** — 파이썬 서버를 못 돌린다.
- 따라서 **앱과 홈페이지를 분리**한다:
  - 앱: Streamlit Community Cloud(무료·GitHub 연동) → `https://…streamlit.app`
  - 홈페이지: 정적 사이트에 제품 소개 + 실행 링크

## 접근 정책: 내부용 (사용자 선택, 2026-07-04)

공개 앱은 누구나 실행 → LLM 비용이 대표 계정에서 소진되고 남용 위험.
그래서 **내부용**으로 결정:
- Streamlit 앱을 **Invite only**(초대 이메일만 열람)로 잠금
- 실행 URL은 공개 HTML에 노출하지 않음 (제품 페이지는 쇼케이스 + 상담 폼 연결)

**Invite only 설정 위치** (Streamlit Community Cloud):
`share.streamlit.io` → My apps → 앱의 `⋮` → **Settings → Sharing 탭**
→ **"Who can view this app"** 드롭다운을 공개("This app is public and searchable")에서
비공개("특정 사용자만")로 변경 → **"Invite viewers by email"**에 허용 이메일(콤마 구분) 입력
→ Save changes. 초대된 이메일로 Google/GitHub 로그인한 사람만 열람 가능.
> ⚠️ 접근권한 변경은 에이전트가 대행 불가(규정) — 사용자가 직접 수행.
> 참고: Netlify(홈페이지)가 아니라 **Streamlit(앱)** 에서 잠근다. 홈페이지는 공개 유지.

## 앱 측 변경 (배포 가능·안전화)

| 파일 | 내용 |
|---|---|
| `.gitignore` | `.env`·`secrets.toml`·서비스계정 JSON·캐시·산출물 커밋 차단 (**키 유출 방지 핵심**) |
| `app.py` | `_bridge_secrets_to_env()` — Streamlit Cloud의 `st.secrets` 값을 `os.environ`으로 전달 (로컬 `.env` 우선). core 모듈은 전부 `os.getenv`라 이 다리 하나로 클라우드 대응 |
| `.streamlit/config.toml` | Iron & Ink 테마(paper/ink/accent) + headless 서버 설정 |
| `.streamlit/secrets.toml.example` | 클라우드 Secrets 입력 템플릿 (실제 secrets.toml은 gitignore) |
| `DEPLOY.md` | 비개발자용 단계별 배포 가이드 (앱→클라우드, 홈페이지→Netlify, 링크 연결) |

- 저장소: GitHub repo `0204yong/anfilt-research-agent` (branch `main`).
  키는 gitignore로 제외 확인 완료(`*.json` 전면 차단 + launch.json만 화이트리스트).
  **처음엔 비공개로 푸시했으나, Streamlit Cloud가 비공개 repo를 못 찾아 결국 공개(public)로 전환함**
  (코드에 키 없음 확인 후). 공개로 유지 권장 — 다시 비공개로 돌리면 Streamlit이 못 찾을 수 있음.
  앱 접근 제한은 저장소 공개여부가 아니라 위 "Who can view this app"(Invite only)로 통제.
- Python 버전: 로컬 3.14 → **클라우드도 3.14로 배포 성공**(3.14.6, 의존성 전부 설치됨).
  당초 3.12/3.13 권장으로 적었으나 Streamlit Cloud 기본값이 3.14이고 정상 빌드됨.

## 홈페이지 측 변경 (`concept-c.html` = 배포본 `dist/index.html`)

정적 사이트, 프레임워크 없음. JS는 제너릭(내비 토글·reveal 관찰자·스크롤스파이 없음)이라
**순수 HTML/CSS 추가만으로 안전**하게 통합.

- 내비게이션: `소개 · [제품] · AI 뉴스 · 문의 게시판 · 무료 상담` — `#product` 링크 추가
  - 라벨은 한글 내비와 통일해 "제품", 섹션 상단 eyebrow는 "ANFILT Product"
- 새 섹션 `#product` (밴드 "잘 벼린 도구 하나가…" 뒤에 배치 → 서사: 도구 언급→도구 공개)
  - 제품 카드: 좌 정체성(Research Agent) + 우 기능 6개 그리드
  - 처리 흐름 4단계: 조사 → 토론 → 종합 → 보고서
  - CSS는 기존 토큰·`.ncard` border 패턴 재사용 (리플로우 안전, 920/680 반응형 검증)
- 실행 링크: 파일 끝 독립 스크립트의 `var APP_URL = ""` 한 줄로 제어
  - 비움 → 버튼 "도입 문의" → 상담 폼(#contact) (공개 안전 기본값)
  - 채움 → 버튼 "Research Agent 실행 ↗" → 내부 앱 링크
  - 홈페이지 기존 철학(CONFIG 키 채우면 동작, 비우면 폴백)과 동일

## 실제 배포 실행 (2026-07-04)

- **홈페이지**: 사용자가 `dist/` 드래그앤드롭으로 Netlify 재배포 →
  anfilt-homepage.netlify.app 라이브에 제품 섹션 반영 확인(curl로 마커 검증).
- **앱**: `claude-in-chrome` 확장 연결 후 에이전트가 브라우저를 조작해 배포 진행.
  - Streamlit 로그인(GitHub) → Create app → Deploy a public app from GitHub
  - Repository `0204yong/anfilt-research-agent`, Branch `main`, Main file `app.py`
  - Secrets: **API 키 입력은 규정상 에이전트 불가** → 사용자가 직접 붙여넣음
  - Deploy → Python 3.14.6 빌드 성공 → `https://anfilt-research-agent-qgb4tpo5e2dvckjh6pmpmi.streamlit.app/`
  - 사이드바에서 **Claude·Gemini 키 인식 확인**(secrets 브리지 정상 동작)
- 앱 URL: `https://anfilt-research-agent-qgb4tpo5e2dvckjh6pmpmi.streamlit.app/`

### 겪은 이슈
- Streamlit이 **비공개 repo를 못 찾음** → repo를 공개로 전환해 해결.
  (OAuth가 public 권한만 있었음. github.com/settings 의 Authorized OAuth Apps에 Streamlit 있으나 private 미승인)
- `.env`가 Windows에서 **AnySign4PC**(보안 프로그램)와 연결돼 더블클릭 시 암호 요구 →
  메모장으로 열어 키 확인하도록 안내(파일 자체는 암호화 안 됨).
- 에이전트가 브라우저 자동배포는 가능하나, **API 키 입력·접근권한 변경은 규정상 불가** →
  해당 단계만 사용자가 수행.

## 남은 작업 (선택)

1. 팀원 접속 필요 시 Sharing에서 이메일 추가 (콤마 구분) 후 Save.

3. 채팅 노출된 Gemini·Claude 키 재발급 권장.
4. `DEPLOY.md`(볼트 밖)에 배포 전 기준 안내 일부 잔존 — 필요 시 최신화.
