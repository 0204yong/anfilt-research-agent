# 10 배포와 홈페이지 연동

회사 홈페이지(anfilt-homepage.netlify.app)에 Research Agent를 **제품으로 등록**하고
실행 가능하게 만드는 작업. 실무 절차는 저장소 루트 [[../DEPLOY]](`DEPLOY.md`) 참고,
이 문서는 설계 배경과 구조를 기록한다.

## 핵심 제약: Streamlit ≠ Netlify

- Research Agent는 **파이썬(Streamlit) 서버 앱** — 지속 실행되는 WebSocket 서버가 필요.
- Netlify는 **정적 호스트** — 파이썬 서버를 못 돌린다.
- 따라서 **앱과 홈페이지를 분리**한다:
  - 앱: Streamlit Community Cloud(무료·GitHub 연동) → `https://…streamlit.app`
  - 홈페이지: 정적 사이트에 제품 소개 + 실행 링크

## 접근 정책: 내부용 (사용자 선택, 2026-07-04)

공개 앱은 누구나 실행 → LLM 비용이 대표 계정에서 소진되고 남용 위험.
그래서 **내부용**으로 결정:
- Streamlit 앱을 **Invite only**(초대 이메일만 열람)로 잠금
- 실행 URL은 공개 HTML에 노출하지 않음 (제품 페이지는 쇼케이스 + 상담 폼 연결)

## 앱 측 변경 (배포 가능·안전화)

| 파일 | 내용 |
|---|---|
| `.gitignore` | `.env`·`secrets.toml`·서비스계정 JSON·캐시·산출물 커밋 차단 (**키 유출 방지 핵심**) |
| `app.py` | `_bridge_secrets_to_env()` — Streamlit Cloud의 `st.secrets` 값을 `os.environ`으로 전달 (로컬 `.env` 우선). core 모듈은 전부 `os.getenv`라 이 다리 하나로 클라우드 대응 |
| `.streamlit/config.toml` | Iron & Ink 테마(paper/ink/accent) + headless 서버 설정 |
| `.streamlit/secrets.toml.example` | 클라우드 Secrets 입력 템플릿 (실제 secrets.toml은 gitignore) |
| `DEPLOY.md` | 비개발자용 단계별 배포 가이드 (앱→클라우드, 홈페이지→Netlify, 링크 연결) |

- 저장소: GitHub **비공개** repo `0204yong/anfilt-research-agent` (branch `main`)로 푸시.
  키는 gitignore로 제외 확인 완료(`*.json` 전면 차단 + launch.json만 화이트리스트).
- Python 버전: 로컬 3.14 → 클라우드는 **3.12/3.13 지정 권장** (신버전 휠 미제공 대비).

## 홈페이지 측 변경 (`concept-c.html` = 배포본 `dist/index.html`)

정적 사이트, 프레임워크 없음. JS는 제너릭(내비 토글·reveal 관찰자·스크롤스파이 없음)이라
**순수 HTML/CSS 추가만으로 안전**하게 통합.

- 내비게이션: `소개 · [제품] · AI 뉴스 · 문의 게시판 · 무료 상담` — `#product` 링크 추가
  - 라벨은 한글 내비와 통일해 "제품", 섹션 상단 eyebrow는 "ANFILT Product"
- 새 섹션 `#product` (밴드 "잘 벼린 도구 하나가…" 뒤에 배치 → 서사: 도구 언급→도구 공개)
  - 제품 카드: 좌 정체성(Research Agent) + 우 기능 6개 그리드
  - 처리 흐름 4단계: 조사 → 토론 → 종합 → 보고서
  - CSS는 기존 토큰·`.ncard` border 패턴 재사용 (리플로우 안전, 920/680 반응형 검증)
- 실행 링크: 파일 끝 독립 스크립트의 `var APP_URL = ""` 한 줄로 제어
  - 비움 → 버튼 "도입 문의" → 상담 폼(#contact) (공개 안전 기본값)
  - 채움 → 버튼 "Research Agent 실행 ↗" → 내부 앱 링크
  - 홈페이지 기존 철학(CONFIG 키 채우면 동작, 비우면 폴백)과 동일

## 검증 (2026-07-04)

- 데스크톱·모바일(375px) 실제 렌더 스크린샷 확인 — 가로 오버플로 0, 테두리 정상
- 태그 균형(섹션 8/8, 스크립트 3/3), 내비/섹션/버튼 DOM 확인
- 추적 파일 전체 키 패턴 스캔 → 실제 키 없음, `.env` gitignore 확인

## 남은 작업 (사용자 몫)

1. `share.streamlit.io`에서 앱 배포 + Invite only 설정 → 앱 URL 획득
2. (내부 실행을 홈페이지에 노출할 경우) `concept-c.html`의 `APP_URL`에 붙여넣기
3. `dist/`를 Netlify에 재배포 (드래그앤드롭 또는 `deploy.js` + NF_TOKEN)
