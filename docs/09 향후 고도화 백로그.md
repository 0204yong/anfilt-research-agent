# 09 향후 고도화 백로그

우선순위는 ★(높음) ★★(중간) ★★★(아이디어) 기준.

## ★ 즉시 가능한 것 (키/크레딧만 있으면)

- [ ] Anthropic 크레딧 충전 → Claude 포함 전체 파이프라인 실검증
  (토론 품질, 구조화 출력 종합, pause_turn 웹검색 재개)
- [ ] `OPENAI_API_KEY` 등록 → GPT 경로 검증 (Responses API tools 호환 확인)
- [ ] Gamma/Canva/Google Slides 키 등록 → 엔진 실검증
  - Gamma: PPTX export 응답 필드명 확인 (`_download_export` 방어 로직 조정)
  - Canva: 토큰 4시간 만료 → **리프레시 토큰 자동 갱신** 구현
  - Slides: 서비스 계정 권한/공유 정책 확인

## ★★ 기능 고도화

- [ ] **엔진 자동 추천 라우터** — 보고서 특성(디자인 중심/데이터 중심/
  협업 필요/민감도)을 LLM이 분류해 엔진 추천, 사용자 오버라이드
- [ ] **회사 PPT 템플릿 적용** — 로컬 빌더가 사용자 .pptx 템플릿(마스터
  슬라이드)을 읽어 브랜드 서식으로 생성
- [ ] **조사 이력 저장** — 실행 결과(JSON+파일)를 로컬 폴더에 아카이브,
  과거 조사 재열람/재활용 UI
- [ ] **스캔 PDF OCR** — 텍스트 없는 첨부 PDF 처리 (예: RapidOCR, 또는
  LLM 비전 입력으로 페이지 이미지 전달)
- [ ] **토론 심판 모드** — 토론 라운드 사이에 진행자가 쟁점 정리·추가 질문을
  던지는 소크라테스식 진행 (현재는 자유 교차 검토)
- [ ] **비용 대시보드** — 프로바이더별 토큰 사용량/예상 비용 표시
  (각 SDK 응답의 usage 필드 수집)
- [ ] **CLI 모드** — pipeline이 이미 Streamlit 독립적이므로
  `python -m core.cli "주제" --pages 12 --out ppt,word` 형태 래퍼만 추가

## ★★★ 장기 아이디어

- [ ] Beautiful.ai / Genspark / Skywork — 공개 API 출시 시 PPTEngine 추가
- [ ] MCP 클라이언트 내장 — 앱이 MCP 호스트가 되어 신규 도구 플러그인식 연결
- [ ] 프로바이더 확장 — Grok, Mistral, 국내 모델(HyperCLOVA X 등)
- [ ] 조사 범위 분할 — 큰 주제를 하위 주제로 쪼개 LLM별 분담 후 통합
  (현재는 전원이 같은 주제 전체를 조사)
- [ ] 보고서 다국어 출력 (영문 고객사 대응)
- [ ] 팀 공유 — Streamlit Cloud/사내 서버 배포 + 접근 제어

## 알려진 기술 부채

- Canva 토큰 수동 갱신 (위 ★ 참고)
- Gamma API 응답 필드는 베타 — 버전업 시 폴링/export 파싱 재확인
- `webfetch`는 JS 렌더링 페이지(SPA) 본문을 못 가져옴 — 필요 시
  Playwright 도입 검토
- OpenAI Chat Completions 폴백 경로는 웹 검색 미지원 — 안내만 하고 있음
