## 공개 배포 시 비밀번호 게이트 (`_require_password`, app.py)

앱을 Streamlit에서 "Anyone with the link"(공개)로 열 때 **아무나 소유자 API 키로
비용을 쓰는 것**을 막는 한 겹 잠금.

- `APP_PASSWORD`(환경변수 또는 Streamlit Secrets)가 설정되면 진입 시 비밀번호를 요구,
  세션당 1회 통과(`st.session_state["_auth_ok"]`). 비교는 `hmac.compare_digest`(UTF-8 바이트).
- **미설정이면 앱을 열지 않고 안내만** 표시(fail-closed) — 공개 전환 중 실수로 비밀번호를
  안 넣어도 무방비 노출되지 않는다. 그래서 **공개 전 순서**: ①Secrets에 APP_PASSWORD 저장
  → ②Sharing을 공개로 전환.
- `st.set_page_config` 직후 호출, 미인증이면 `st.stop()`으로 이하(사이드바·조사) 실행 차단.
- 내부용(초대 전용)으로만 쓸 거면 APP_PASSWORD를 비워도 되지만, 그 경우 앱은 잠긴 안내만
  뜨므로 Streamlit Sharing의 초대 방식과 병행하려면 비밀번호를 설정해 두는 편이 낫다.

## 실행


개발용 미리보기 설정은 `.claude/launch.json`에 있음 (포트 8501).

## 실행당 비용 구조

LLM 호출 수 = **참여 LLM 수 × (1 + 토론 라운드) + 1(종합)**
- 예: 3개 LLM, 토론 1라운드 → 7회 호출
- 레퍼런스 자동 탐색은 별도 1회 (웹 검색 포함)
- 웹 검색은 프로바이더별 추가 요금 가능성 있음
- Gamma는 생성당 크레딧 차감

## 테스트

세션 스크래치에 테스트 스크립트가 있었음 (임시 폴더라 소멸 가능).
핵심 검증 항목은 [[08 작업 이력]]에 기록되어 있으므로, 회귀 테스트가
필요하면 다음을 다시 만들면 된다:
1. 보고서 3종 빌더 — 더미 REPORT_SCHEMA JSON으로 생성 + 재오픈 확인
2. `_length_plan` 1~20장 합계 = 목표 장수
3. `parse_json_loose` — 코드펜스/잡음/배열 케이스
4. `extract_file_text` — 형식별 샘플 파일 생성 후 추출
5. (키 필요) 프로바이더별 generate / 파이프라인 e2e
