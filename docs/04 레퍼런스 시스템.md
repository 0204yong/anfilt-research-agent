# 04 레퍼런스 시스템

레퍼런스를 넣는 경로는 3가지이며, 최종적으로 전부
`brief.reference_texts` (dict: 라벨 → 본문 텍스트)로 합쳐져
**모든 LLM에게 동일한 원문**으로 제공된다.

## ① URL 직접 입력 — `core/webfetch.py`

- `requests` + `BeautifulSoup` 로 본문 추출
- script/style/nav/footer 등 제거 후 텍스트만
- URL당 최대 20,000자 (초과 시 절단 표시)
- PDF URL은 미지원 안내 반환 / 실패 시 `[가져오기 실패: ...]`
- 브라우저 User-Agent 헤더 사용 (봇 차단 회피)

## ② 자동 탐색 — `core/discovery.py`

주제만 입력하면 LLM이 웹 검색으로 출처를 발굴.

- 프롬프트 우선순위: 공식 기관·규제기관 → 표준/프레임워크 발행기관(ISSB,
  GRI, CDP, SBTi 등) → 국제기구·연구기관 → 전문 언론
- 규칙: 실존 확인된 페이지만, 리다이렉트/단축 URL 금지, 사이트당 최대 2개
- 출력: `{"references": [{title, url, publisher, reason}]}`
  - 검색 도중 JSON이 깨지면 → 재포맷 요청 1회
  - URL http 검증 + 중복 제거 후 최대 count개
- **폴백**: `discover_with_fallback()` — Claude → Gemini → GPT 순서로
  시도, 실패하면 다음으로 (전부 실패 시 원인 목록과 함께 RuntimeError)
- UI: 찾은 출처가 체크박스로 표시 → 체크된 것만 조사 시작 시
  URL 목록에 병합 (세션 키 `ref_sel_{i}`, 재탐색 시 초기화)

검증 사례: "EU CBAM" → EU 집행위 공식 페이지·산업부 가이드라인·KIEP·KITA,
"중소기업 Scope 3" → 환경부 안내서·국립환경과학원 등 발굴 확인.

## ③ 파일 첨부 — `core/filerefs.py`

`st.file_uploader` (다중 파일) → `extract_file_text(filename, bytes)`

| 형식 | 라이브러리 | 추출 방식 |
|---|---|---|
| pdf | pypdf | 페이지별 텍스트, `[p.N]` 접두 |
| docx | python-docx | 문단 + 표 (셀을 `\|`로 연결) |
| pptx | python-pptx | 슬라이드별 텍스트 상자 + 표 |
| xlsx/xlsm | openpyxl (read_only, data_only) | 시트별, 시트당 200행 제한 |
| txt/md/csv | 내장 | utf-8 → cp949 → replace 순 디코딩 |

- 파일당 최대 30,000자
- 스캔 이미지 PDF 등 텍스트 없음 → 안내 문자열 (OCR은 백로그 → [[09 향후 고도화 백로그]])
- 키는 `[첨부 파일] {파일명}` 형태로 reference_texts에 저장

## 브리프 조립 (`pipeline._brief_block`)

```
## 조사 주제
## 검색 키워드          ← 있으면 web_search=True 트리거
## 레퍼런스 자료 (전체 원문 발췌)
   ### 출처: {URL 또는 [첨부 파일] 이름}
## 추가 지시사항
```
