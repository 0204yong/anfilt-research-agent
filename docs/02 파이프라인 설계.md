# 02 파이프라인 설계

파일: `core/pipeline.py`

## 데이터 모델

| 클래스 | 역할 |
|---|---|
| `ResearchBrief` | topic, keywords, reference_urls, reference_texts{url:본문}, instructions, persona |
| `AgentFinding` | 조사 결과 — provider_key/label, model, text, error |
| `DiscussionTurn` | 토론 발언 — round_no, provider, text, error |
| `PipelineResult` | findings, discussion, report(JSON), moderator_label |

## ① 조사 `run_research()`

- 전 프로바이더 병렬 (`ThreadPoolExecutor`)
- **키워드가 있으면** `web_search=True` — 각 LLM의 자체 검색 도구 사용
- 프롬프트 구조: 브리프 블록 + 출력 형식 지시
  (핵심 요약 / 주요 발견 / 세부 분석 / 데이터·수치 / 출처 목록)
- 실패는 `AgentFinding.error`에 기록하고 계속 진행

## ② 토론 `run_discussion()`

- 성공한 프로바이더가 **2개 이상**일 때만 진행 (아니면 빈 리스트)
- 라운드마다: 각자 [자신의 최신 입장 + 동료들의 최신 입장(익명)]을 받아
  1) 동의/이견 2) 보완 3) **수정된 최종 입장** 을 작성
- `current` dict가 라운드마다 갱신 → 다음 라운드 입력
- 프롬프트에 "근거 없는 양보 금지" 명시 (동조 편향 방지)

## ③ 종합 `run_synthesis()`

- 진행자 우선순위: `pick_moderator()` — Claude(구조화 출력 지원) → GPT → Gemini
- 단, 조사에 성공한 프로바이더 중에서만 선택
- 종합 방식 2가지 (UI 라디오):
  - `synthesize` — 교차 검증 통합 (공통 확인 내용 우선, 상충 시 근거 강한 쪽)
  - `best` — 가장 우수한 결과 중심 + 검증된 보완만 반영
- 결과는 `REPORT_SCHEMA` JSON:
  - Claude: `output_config.format` 구조화 출력 (스키마 보장)
  - GPT/Gemini: 프롬프트 지시 + `parse_json_loose()` (코드펜스/잡음 방어 파싱)

## REPORT_SCHEMA (보고서 JSON)

```json
{
  "title": "...", "subtitle": "...",
  "executive_summary": "...",
  "key_findings": ["..."],
  "sections": [{"heading", "content", "bullets": []}],
  "data_tables": [{"title", "headers": [], "rows": [[]]}],
  "recommendations": ["..."],
  "sources": [{"title", "url"}]
}
```

- 모든 객체에 `additionalProperties: false` + 전 필드 required
  (Claude 구조화 출력 요구사항)
- 표의 셀은 전부 문자열 (렌더링 단순화)

## 분량 계획 `_length_plan(target_pages)`

목표 PPT 장수(1~20)를 받아 구성 요소 개수를 역산 — [[05 보고서 생성]] 참고.

| 구간 | PPT 구성 | 산식 |
|---|---|---|
| 1장 | 원페이저 | sections=0, tables=0, 요약 3~4문장 |
| 2~7장 | [표지+요약] + 섹션/표 + [발견+제언] | sections + tables = target − 2 |
| 8~20장 | 표준 5장 고정 + 섹션/표 | sections + tables = target − 5 |

서술 깊이(섹션당 글자 수, bullets 수)도 구간별로 지시 →
Word 분량도 비례해서 조절된다. 진행자 프롬프트에 "반드시 준수"로 삽입.

## 실행 진입점

```python
run_pipeline(providers, brief, rounds=1, mode="synthesize",
             target_pages=12, on_update=콜백) -> PipelineResult
```

`on_update(str)`는 UI 진행 표시용 — Streamlit 밖(배치/CLI)에서도 사용 가능하게
파이프라인은 Streamlit에 의존하지 않는다.
