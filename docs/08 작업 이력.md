## 2026-07-04

### 1. 초기 구축
- 요구사항: 다수 LLM이 동일 레퍼런스/키워드로 동시 조사 → 상호 토론 →
  우수 내용 선별/종합 → PPT·Word·Excel 보고서 생성
- 결정: **멀티 프로바이더**(Claude+GPT+Gemini, 사용자 선택) + **Streamlit UI**
- 구현: config / providers 3종 / pipeline(조사·토론·종합) / reports 3종 / app.py
- 검증: 보고서 빌더 스모크 테스트, UI 렌더링 확인

### 2. Gemini 키 등록 + 버그 수정
- `GOOGLE_API_KEY` 등록, 기본·웹검색 그라운딩 검증
- **버그**: 짧은 max_tokens에서 빈 응답 → thinking 토큰 차감이 원인
  → 출력 하한 8192 + `_extract_text()` 방어 로직 (→ [[03 LLM 프로바이더]])
- Gemini 단독 e2e (조사→종합→3종 파일) 성공

### 3. 레퍼런스 자동 탐색 기능
- `core/discovery.py` — LLM 웹 검색으로 신뢰 출처 발굴, 체크박스 선택 후
  조사에 병합. `parse_json_loose` 배열 지원 확장
- 실검증: EU CBAM 주제 → EU 집행위·산업부·KIEP·KITA 등 발굴

### 4. Anthropic 키 등록 + 폴백 보강
- 키 유효 확인, 단 **크레딧 잔액 0** (`credit balance too low` 400)
- `discover_with_fallback()` 추가 — 탐색 실패 시 다음 프로바이더로 자동 전환
- 실검증: Claude 실패 → Gemini 폴백, 조사 단계 오류 격리, 진행자 자동 재선택

### 5. 보고서 분량 설정
- 사이드바 슬라이더 (처음 8~20장 → 이후 **1~20장** 확장)
- `_length_plan()` — 장수 역산 (원페이저/컴팩트/표준 3계층)
- PPT 빌더에 원페이저·컴팩트 레이아웃 추가
- 검증: 1~20 전 구간 정합 + 실행 테스트 (10장→10장, 1장→원페이저 1장)

### 6. 파일 첨부 레퍼런스
- `core/filerefs.py` — pdf/docx/pptx/xlsx/txt/md/csv 본문 추출 (pypdf 추가)
- 형식별 샘플 생성 테스트 전부 통과 (cp949 CSV, 한글 표 포함)

### 7. 외부 PPT 디자인 엔진 (키 대기 스캐폴드)
- 사전 조사: Gamma(공식 API+MCP)·Canva(공식 MCP+Connect API)·
  Google Slides(공식 API)는 연동 가능 / Beautiful.ai·Genspark·Skywork는
  공개 API 없어 보류 (→ [[06 PPT 디자인 엔진]])
- `core/ppt_engines/` — PPTEngine 추상화 + 3개 엔진 구현
  - Gamma: Generate API (마크다운 카드 → 디자인 덱 + PPTX)
  - Canva: Design Import API (로컬 PPTX → 편집 가능한 Canva 디자인)
  - Google Slides: 서비스 계정 + batchUpdate (공유 링크)
- UI: 다운로드 탭에 엔진 선택·실행·결과 링크, 키 없으면 발급 안내
- 검증: 레지스트리/가용성/마크다운 변환 테스트 (실API는 키 등록 후)

### 8. 설계서 볼트 생성 (이 문서)
