    is_error `'['` 관례·is_attachment — 기밀 격리 3단계 대비)
  - 파일명 정규화: Windows 금지문자·예약명(CON 등)·꼬리 점, zip은 UTF-8 플래그 자동
- `app.py`: 실행 성공 시 `run_ctx`(brief+파라미터+executed_at) 캡처(설계서가 지적한
  PipelineResult 누락 보완), 다운로드 탭에 "🗂 지식볼트 zip 다운로드" 추가
- 검증: 더미 파이프라인 결과 스모크 테스트 전 항목 통과(zip 구조·frontmatter YAML
  파싱·이스케이프·프로비넌스·예약명), `py_compile` 통과

### 18. 지식볼트 2단계 — Supabase 아카이브+재열람 구현 ✅ (활성화는 사용자 설정 대기)
- `supabase-ra-runs-setup.sql` — `ra_runs` 테이블(run_id PK, jsonb record,
  schema_version) + RLS 전면 차단(anon/authenticated) — 접근은 service_role만
- `core/store.py` (Streamlit 비의존) — PostgREST REST 직접 호출(신규 의존성 0):
  save_run(업서트)/list_runs/load_run/record_to_state(필드 명시 선별 복원)
- `app.py` — 성공 직후 자동 저장(`_saved_run_id` 중복 방지), 실패 시 경고+재시도+
  zip 폴백 안내(안전장치 4), 사이드바 "🗂 이전 조사"(60초 캐시)→5탭 UI 복원,
  run_id를 zip·아카이브가 공유, `.env.example`에 SUPABASE 2항목 추가
- 검증: 왕복 복원·REST 모킹(업서트 헤더·오류 노출·빈 결과)·여분 키 내성·
  run_id 공유 테스트 전부 통과. UI 부팅 검증 — 키 미설정 시 안내 캡션만, 예외 없음
- **라이브 검증 완료 (같은 날)**: 사용자가 SQL 실행(처음에 파일명만 붙여넣는
  실수 → 본문 붙여넣기로 해결) + .env에 키 입력(에이전트가 빈 항목 추가 후
  메모장 열어 사용자가 키만 기입 — 키는 채팅에 노출 안 함).
  CLI 왕복(저장→중복 없음→로드→복원→삭제) + 앱 e2e(실조사→자동 저장→새 세션
  불러오기→5탭 복원→볼트 zip) 전부 실측 통과
- 버그 픽스: 사이드바(상단 렌더)가 저장(하단 실행) 결과를 못 보던 문제 →
  저장 성공 직후 `st.rerun()` + 토스트 대신 지속 캡션("🗂 이력 저장됨")
- 배포 앱 활성화 남음: push + Streamlit Secrets에 SUPABASE 2줄(사용자)
