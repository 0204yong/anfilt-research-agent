## PPT — `pptx_builder.py`

`build_pptx(report, target_pages=None)` — 분량에 따라 3종 레이아웃 자동 선택:

### 표준 (8~20장 또는 target_pages=None)
표지 → Executive Summary → 핵심 발견사항 → 섹션들(bullets 우선, 없으면 content)
→ 데이터 표들(행 14개 제한) → 제언 → 출처

### 컴팩트 (2~7장)
[표지+요약 통합] → 섹션들 → 표들 → [핵심발견+제언 통합]

### 원페이저 (1장)
상단 제목 밴드 + 좌측 Executive Summary / 우측 핵심발견·제언 2단 구성

### 장수 정합성
`_length_plan()`이 구성 개수를 역산하므로 **생성 장수 = 목표 장수**.
1~20장 전 구간 + 실제 LLM 실행(1장, 10장)으로 검증 완료.

## Word — `docx_builder.py`

- 표지(제목/부제) → Executive Summary → 핵심 발견사항(번호 목록) →
  섹션(Heading 1 + 본문 + 불릿) → 데이터 표(Light Grid Accent 1) →
  제언 → 출처
- **한글 폰트 주의**: python-docx에서 한글은 `w:rFonts`의 `eastAsia`
  속성까지 지정해야 적용됨 → `_set_korean_font()` 헬퍼 참고

## Excel — `xlsx_builder.py`

| 시트 | 내용 |
|---|---|
| 요약 | 제목/부제 + Executive Summary + 핵심 발견 + 제언 |
| 본문 | 섹션 표 (섹션명/내용/요점 3열) |
| (표 제목별) | data_tables 각각 별도 시트, 헤더 스타일 + 테두리 |
| 출처 | 제목/URL — URL은 하이퍼링크 |

- 시트명은 Excel 금지문자 제거 + 28자 제한 + 중복 회피 (`_safe_sheet_name`)
- 열 너비 자동 조정 (`_autofit`, 최대 60)

## 파일 생성 시점 (app.py)

- 파이프라인 완료 시가 아니라 **다운로드 탭 최초 렌더링 시 1회 생성**
  → `st.session_state["files"]`에 캐시 (download_button 클릭 rerun 대비)
- 파일명: `{제목 안전화}_{날짜}.{확장자}`
